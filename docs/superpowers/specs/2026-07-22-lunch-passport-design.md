# 점심 여권 설계

## 목표

직원이 완료 방문한 식당을 개인 도감 안에서 한눈에 확인할 수 있도록, 점심 여권 요약과 카테고리별 수집 현황을 제공한다.

## 범위

- `/collection` 상단에 점심 여권 요약을 표시한다.
- 개인 완료 방문은 `visits`의 완료 기록, 완료된 동행의 방장 기록, 완료된 동행 참여 기록을 모두 포함한다.
- 활성 식당만 수집 대상이다.
- 전체 활성 식당 수 대비 방문한 고유 식당 수와 완료율을 표시한다.
- 카테고리마다 전체 식당 수, 방문한 고유 식당 수, 완료율을 표시한다.
- 방문한 식당 항목에는 식당명, 방문 횟수, 첫 방문일, 최근 방문일을 표시하고 식당 상세로 연결한다.
- 방문하지 않은 카테고리는 '아직 방문 전' 안내를 표시한다.
- 기존 도감의 필터·즐겨찾기·전체 목록은 그대로 유지한다.

## 집계 규칙

1. 개인 완료 방문은 식당별로 모두 누적하되, 완료율의 방문 식당 수는 고유 식당 수로 계산한다.
2. 개인 방문의 날짜는 `visits.visit_date`를 사용한다.
3. 동행의 날짜는 `appointments.scheduled_at`을 Asia/Seoul 날짜로 변환해 사용한다.
4. 첫 방문일은 가장 이른 완료 방문일, 최근 방문일은 가장 늦은 완료 방문일이다.
5. 카테고리와 전체 분모는 활성 식당만 포함한다.
6. 완료 방문이 하나도 없으면 전체 완료율은 0%이며 모든 카테고리는 미방문 상태다.

## 구현

- 데이터베이스 스키마는 변경하지 않는다.
- 서버 전용 조회가 활성 식당과 개인의 완료 방문 원천을 가져온 뒤 순수 집계 함수가 여권 결과를 만든다.
- `/collection` 페이지는 기존 `getCollectionData` 결과와 새 여권 결과를 함께 조회한다.
- 여권 카드 UI는 도감 페이지 전용 컴포넌트로 분리한다.

## 제외 범위

- 배지·보상·레벨·시즌제, 수동 편집, 방문 사진, 과거 기간 탐색, 비활성 식당 표시.
- 추천 로직이나 방문·동행·리뷰 저장 방식의 변경.

## 검증

- 순수 집계 함수에서 개인·방장·참여 방문 합산, 고유 식당 완료율, 첫·최근 방문일, 비활성 식당 제외, 빈 상태를 단위 테스트한다.
- 여권 카드에서 방문 카테고리·미방문 카테고리·식당 상세 링크를 테스트한다.
- 도감 페이지에서 여권 노출과 빈 상태를 테스트한다.
