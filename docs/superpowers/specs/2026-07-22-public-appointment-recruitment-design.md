# 공개 동행 모집 설계

## 목표

방장이 식당과 시간을 정한 동행을 공개 모집으로 만들고, 다른 직원의 참여 신청을 승인 또는 거절할 수 있게 한다.

## 범위

- 약속 생성 시 공개 모집 여부와 정원을 입력한다.
- 정원의 기본값은 4명이고, 방장은 2명부터 10명 사이에서 선택한다.
- 방장 자신도 정원에 포함한다. 정원이 4명이면 참가자 3명을 승인할 수 있다.
- 공개·활성·미래 약속은 홈에서 참여 가능한 동행으로 표시한다.
- 신청은 기존 참가자 행의 `pending` 상태를 사용하고, 방장이 `accepted` 또는 `declined`로 결정한다.
- 승인, 거절, 취소는 기존 알림 체계로 알린다.

## 데이터 모델

`appointments`에 다음 필드를 추가한다.

- `is_public boolean not null default false`
- `capacity smallint` — 공개 약속은 2~10, 비공개 약속은 null

공개 약속의 정원 검증은 서버 액션에서 수행한다. 데이터베이스 제약은 공개/비공개 필드의 일관성과 범위를 보장한다.

## 동작

1. 방장이 약속 생성 화면에서 공개 모집을 선택하면 정원 입력이 나타난다.
2. 공개 약속은 홈의 모집 목록에 보인다. 방장, 이미 신청한 직원, 종료·취소·정원 충족 약속은 신청 대상에서 제외한다.
3. 직원이 신청하면 `appointment_participants`에 `pending` 행을 만든다.
4. 방장은 약속 상세에서 대기 신청자를 승인 또는 거절한다. 승인 직전에 현재 승인 인원과 방장을 합쳐 정원을 다시 확인한다.
5. 승인 인원이 정원에 도달하면 추가 신청과 대기 신청자 승인을 막는다. 기존 대기 신청자는 유지한다.

## 예외와 보안

- 모든 생성·신청·승인·거절은 직원 세션과 약속 상태를 서버에서 확인한다.
- 방장만 신청자를 결정할 수 있고, 방장은 자기 약속에 신청할 수 없다.
- 동시 신청 또는 동시 승인은 업데이트 결과와 정원 재확인으로 중복 승인을 막는다.
- 비공개 초대 동행의 기존 응답 흐름은 바꾸지 않는다.

## 제외 범위

- 자동 승인, 대기열 승격, 공개 댓글·채팅, 정원 변경 후 자동 조정은 구현하지 않는다.

## 검증

- 정원 입력 및 공개 모집 검증을 단위 테스트한다.
- 생성 폼과 홈 모집 목록의 표시 조건을 컴포넌트 테스트한다.
- 서버 액션은 권한, 만료, 중복 신청, 정원 도달을 확인한다.
