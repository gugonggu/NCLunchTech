# Kakao 식당 수집 범위 및 상호명 추가 설계

## 목표

회사 주변의 밀집 지역에서 누락되는 Kakao 식당을 줄이고, 관리자가 특정 상호명을 찾아 선택 등록할 수 있게 한다.

## 범위

- Kakao 카테고리 동기화는 회사 중심 2km를 400m 간격의 고정 격자로 검색한다.
- 각 격자 지점은 400m 반경으로 음식점(FD6)과 카페(CE7)를 검색한다.
- Kakao 장소 ID로 전체 결과를 중복 제거한 뒤 기존 저장 규칙으로 등록한다.
- 관리자 식당 관리 화면에서 상호명을 검색하고, 회사 2km 이내 Kakao 결과 하나를 선택해 등록할 수 있다.
- 동일 Kakao 장소 ID는 새로 등록하지 않고 기존 식당으로 안내한다.
- 검색·등록은 관리자 인증 및 관리자 작업 로그를 적용한다.

## 제외 범위

- 주소, 좌표, Kakao 장소 URL을 직접 입력하는 수동 등록
- Kakao API가 결과를 반환하지 않는 장소의 별도 데이터원 보강
- 자동 주기 동기화, 알림, Naver·Google API 연동
- 2km 밖 장소의 등록

## 고정 격자 동기화

현재 2km 범위의 1km 간격·800m 반경 검색 대신, 400m 간격·400m 반경 검색을 사용한다. 넓은 중심 검색이 Kakao의 페이지 처리 가능 결과 수를 초과하는 밀집 지역을 더 작은 검색 영역으로 나눈다. 전체 영역을 항상 같은 방식으로 검색하므로 실행 결과가 예측 가능하다.

검색 지점은 2km 경계 밖 400m까지 포함해 가장자리 식당이 누락되지 않게 한다. API 호출은 순차 실행하고, 이미 받은 장소는 `Map<placeId, place>`로 중복 제거한다. 기존 2km 수집 범위와 식당 저장 스키마는 바꾸지 않는다.

동기화 결과에는 검색 격자 수, 발견 수, 신규 등록 수, 중복 수를 계속 표시한다. 격자가 촘촘해져 호출 수가 증가하므로 관리자 수동 실행만 제공한다.

## 상호명 선택 등록

관리자 식당 관리 화면에 별도 카드로 상호명 입력 폼을 둔다. 서버 전용 Kakao 키워드 검색은 회사 좌표를 중심으로 2km 이내 후보를 반환한다. 각 후보는 이름, 분류, 주소, 회사 기준 거리를 표시한다.

관리자는 하나의 후보를 선택해 등록한다. 서버는 로그인한 관리자를 다시 확인하고, 후보의 Kakao 장소 ID를 이용해 다시 조회한 검색 결과인지 검증한 뒤 기존 `restaurants` 행과 같은 필드를 저장한다. 같은 Kakao 장소 ID가 있으면 삽입하지 않고 해당 식당 관리 상세로 이동한다. 성공·중복 등록 모두 관리자 로그에 기록한다.

## 오류와 제한

- 회사 좌표 또는 Kakao 키가 없으면 기존 동기화와 같은 명확한 오류를 표시한다.
- 빈 검색 결과에는 Kakao Local API에 검색 결과가 없다는 안내를 표시한다.
- 2km 밖 후보는 서버와 UI 양쪽에서 제외한다.
- Kakao 검색 결과의 이름·좌표·장소 ID는 클라이언트가 전송한 값으로 신뢰하지 않고 서버가 장소 ID 기반으로 다시 검증한다.

## 검증

- 고정 400m 격자가 중심·경계 검색 지점을 만들고, 기존 1km 격자보다 많은 지점을 만드는 단위 테스트
- Kakao 키워드 결과의 거리 계산 및 2km 필터 단위 테스트
- 중복 장소 ID의 등록 방지와 관리자 등록 결과 변환 단위 테스트
- 상호명 검색·후보 선택 UI 컴포넌트 테스트
- lint, typecheck, 전체 테스트, build
