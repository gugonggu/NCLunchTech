# 이번 달의 식당 설계

## 목표

직원들의 실제 완료 방문과 리뷰를 바탕으로 매월 한 곳의 `이번 달의 식당`을 선정해 홈과 리더보드에서 소개한다.

## 선정 기준

1. Asia/Seoul 기준 이번 달의 완료 방문 수가 가장 많은 식당을 우선한다.
2. 완료 방문 수가 같으면 해당 월에 작성된 리뷰의 평균 맛 점수가 높은 식당을 우선한다.
3. 평균 맛 점수까지 같거나 리뷰가 없으면 가장 최근 완료 방문 시각이 늦은 식당을 우선한다.
4. 모든 기준까지 같으면 식당 이름 오름차순으로 결정해 결과를 안정적으로 만든다.

완료 방문 또는 해당 월 리뷰가 없는 식당은 후보에서 제외한다. 단일 식당이 가장 많이 방문됐어도 최소 방문 횟수는 두지 않는다.

## 화면

- 홈의 오늘 같이 먹기 아래에 `이번 달의 식당` 카드로 노출한다.
- 카드에는 식당명, 카테고리, 이번 달 완료 방문 횟수, 평균 맛 점수(리뷰가 있을 때만), 선정 이유를 표시한다.
- 카드를 누르면 기존 식당 상세 화면으로 이동한다.
- 리더보드 상단에는 동일한 선정 결과를 간단히 요약해 월간 활동 맥락을 유지한다.

## 구현

- 기존 `visits`, `reviews`, `restaurants`만 조회하며 데이터베이스 스키마는 바꾸지 않는다.
- 서버 전용 쿼리가 서울 월 범위에 맞는 완료 방문과 리뷰를 가져온 뒤, 순수 집계 함수가 순위를 만든다.
- 홈과 리더보드는 같은 결과 객체를 사용한다.

## 제외 범위

- 월별 수상 이력 저장, 수동 선정·제외, 배지 발급, 과거 월 탐색은 이번 범위에 포함하지 않는다.

## 검증

- 방문 수·평균 맛 점수·최근 방문·이름 순의 동률 해소를 단위 테스트한다.
- 후보가 없을 때 카드를 숨기는 홈·리더보드 표시를 테스트한다.
