# 앤시점심기술 — Claude 프로젝트 지침
## 1. 목적
- 앤시정보기술 직원이 KNN타워 주변 식당을 추천받고, 동료와 점심 약속을 만들고, 방문·메뉴·가격·리뷰를 기록하는 사내 웹서비스다.
- 서비스명: 앤시점심기술
- 기준 위치: 부산광역시 해운대구 센텀서로 30, KNN타워
- 실제 사내 사용 가능한 완성도가 목표다.
- 이름 외에는 억지스러운 개발자·기술 콘셉트를 사용하지 않는다.
## 2. 확정 기술 스택
- Next.js App Router, TypeScript, Tailwind CSS
- Supabase PostgreSQL, Storage
- Supabase Auth: 관리자 인증에만 사용
- Supabase Realtime: 2차 투표 기능
- Kakao Local API / Kakao Map
- Vercel 배포
- 일반 직원의 DB 작업은 `브라우저 → Next.js Server Action/Route Handler → 권한 검증 → Supabase`로 처리한다.
- 서비스 역할 키와 Kakao REST API 키는 서버 환경변수에만 저장한다.
- 브라우저에서 Supabase DB를 직접 수정하거나 비밀 키를 노출하지 않는다.
## 3. 디자인
- 모바일 우선 반응형, PC에서는 중앙 정렬 앱형 레이아웃
- 흰색 배경, 따뜻한 주황 포인트: `#F28C28`, 진한 주황 `#D96F12`, 연한 배경 `#FFF4E8`
- 둥글고 친근한 카드·버튼, 카드 radius 16~20px, 버튼 12~16px
- 짧고 담백한 문구, 과한 게임 UI·그림자·애니메이션 금지
## 4. 위치와 식당
- 회사 위치 한 곳만 지원하고 사용자 GPS는 사용하지 않는다.
- 반경: 300m / 500m / 800m / 1.2km / 2km, 기본 800m
- 거리와 도보 시간은 KNN타워 기준이다.
- Kakao 장소 ID를 내부 식당 연결 키로 사용한다.
- Kakao에서 기본 정보만 가져오고 메뉴·가격·영업시간은 자체 DB에서 관리한다.
- 지속적인 네이버 지도 크롤링 등은 하지 않는다.
- 분류: 한식, 중식, 일식, 양식, 분식, 아시아 음식, 패스트푸드, 카페·간단식, 기타
## 5. 직원 인증
- 생성: 회사 공용 초대코드 + 중복 불가 닉네임 + 4자리 PIN + PIN 확인
- 로그인: 닉네임 + 4자리 PIN
- PIN은 강한 단방향 해시로 저장한다.
- 세션은 예측 불가능한 토큰을 사용하고 쿠키는 HttpOnly, Secure, 적절한 SameSite를 적용한다.
- DB에는 세션 토큰 원문이 아닌 해시를 저장한다.
- 마지막 사용 기준 30일 자동 연장, PC·휴대폰 세션은 독립적이다.
- PIN 초기화나 계정 비활성화 시 모든 세션을 폐기한다.
- 로그인 반복 실패를 제한한다.
- 직원 인증에 Supabase Auth를 사용하지 않는다.
## 6. 관리자
- Supabase Auth 이메일 + 비밀번호를 사용하며 직원 인증과 분리한다.
- 초기 관리자 1명, 필요 시 추가 가능
- 관리자 서버 권한 검사와 작업 로그 필수
## 7. 1차 내비게이션과 핵심 흐름
- 하단 메뉴: 홈 / 식당 찾기 / 도감 / 내 정보
- `함께 먹기`는 2차에서 추가한다.
- 핵심 흐름: 로그인 → 즉시 추천 → 여기로 결정 → 혼자/동료와 함께 → 방문 예정/약속 → 1시간 후 방문 확인 → 메뉴·가격 기록 → 리뷰/건너뛰기 → 도감 반영
## 8. 홈과 빠른 추천
- 홈 순서: 서비스명·알림, 인사, 오늘 뭐 먹지?, 조건 추천, 식당 검색, 다가오는 약속, 방문 확인, 최근 방문, 새 식당, 월간 활동
- 데이터가 없는 영역은 숨기고 방문 확인은 우선 노출한다.
- `오늘 뭐 먹지?`는 질문 없이 즉시 추천한다.
- 기본 조건: 800m, 영업 중 우선, 휴무·브레이크 제외, 대기·재료 소진 반영, 최근 방문 감점, 연속 추천 방지, 신규 식당 노출
- 결과: 메인 1곳 + 대안 2곳 + 사실적인 추천 이유 + 메뉴·가격·리뷰 수
- 동작: 여기로 결정 / 지도 / 상세 / 다시 추천
- 다시 추천은 무제한이며 넘긴 식당은 당일 임시 제외한다.
- 제외 목록 복구 가능, 후보 부족 시 안내 후 재포함 가능
## 9. 조건 추천과 식당 찾기
- 목적: 아무거나 / 맛 / 빠른 식사 / 가성비 / 혼밥 / 단체 / 가까운 곳 / 새로운 식당
- 필터: 식당명·메뉴명, 음식 분류, 거리, 메뉴 가격, 영업 중, 최근 방문 제외, 혼잡도
- 개인 기본 예산은 만들지 않는다.
- 등록 메뉴 중 기준 금액 이하 메뉴가 하나라도 있으면 가격 필터에 포함한다.
- 가격이 없으면 `가격 정보 없음`으로 표시한다.
- 식당 찾기는 목록 기본, 지도 전환 가능, 기본 정렬 추천순
- 정렬: 추천순 / 가까운 순 / 리뷰 많은 순 / 신규 / 낮은 메뉴 가격 순
## 10. 식당 상세와 공동 편집
- 표시: 대표 사진·갤러리, 기본 정보·거리·지도, 영업시간·휴무, 메뉴·가격, 평가·리뷰, 상태 제보, 즐겨찾기
- 동작: 여기로 결정, 메뉴 추가, 가격 수정, 판매 종료, 영업시간 수정, 정보 확인, 상태 제보, 사진 등록, 신고
- 등록자·수정자·확인 날짜와 변경 이력을 보존한다.
- 일반 직원은 완전 삭제할 수 없고 관리자는 삭제·복구 가능하다.
- 상태: 한산 / 조금 기다림 / 대기 길음 / 재료 소진 / 오늘 휴무 / 점심 영업 안 함 / 단체 자리 부족 / 위치 오류 / 폐업 추정
- 혼잡도 60분, 재료 소진 영업 종료까지, 오늘 휴무 당일, 폐업 추정은 관리자 확인까지 유지
## 11. 사진
- 음식·외관·메뉴판·동행·장난 사진 등을 자유롭게 등록 가능
- 업로더·날짜 표시, 본인 삭제, 신고, 관리자 삭제, 업로드 최적화
- 대표 사진이 없으면 카테고리 기본 이미지
- 사진 댓글·좋아요·ㅋㅋ 반응은 만들지 않는다.
## 12. 여기로 결정과 약속
- 선택: 혼자 먹기 / 동료와 함께
- 12:30 이전 기본 시간은 오늘 12:30, 이후는 현재 시각 + 30분
- 날짜·시간 수정 가능, 과거 시간 불가
- 동료 초대: 닉네임 직접 선택 + 초대 링크
- 초대된 직원은 자동 참여하지 않고 수락·거절한다.
- 링크는 약속 시작 시간까지만 유효하다.
- 비로그인/미가입 직원은 인증 후 원래 초대 화면으로 복귀한다.
- 상태: 대기 / 확정 / 거절 / 응답 없음 / 취소
- 방장 메모 최대 100자, 채팅 없음
- 방장만 전체 취소·식당·시간·메모 변경 가능, 참여자는 본인만 불참 가능
## 13. 방문 확인과 리뷰
- 예정 시간 1시간 후 방문 확인 대기로 전환한다.
- 선택: 다녀왔어요 / 가지 않았어요 / 나중에 할게요
- 참여자별로 개별 확인하며 방장의 확인이 다른 사람에게 영향을 주지 않는다.
- 다녀왔어요: 방문 완료, 실제 메뉴·가격 기록, 리뷰 안내, 도감·통계 반영
- 리뷰 없이 방문만 완료할 수 있다.
- 리뷰는 사용자당 식당별 1개이며 재작성 시 수정한다.
- 필수 1~5점: 맛 / 제공 속도 / 가격 만족도 / 혼밥 적합성
- 재방문: 다시 갈래요 / 고민 / 다시 가지 않을래요
- 선택: 양 / 혼잡 / 단체 적합성 / 청결 / 태그 / 한 줄 후기
- 리뷰 댓글 없음, `도움돼요`는 2차
## 14. 1차 도감
- 전체 식당, 방문/미방문, 리뷰 작성, 즐겨찾기, 음식 분류별 현황, 상세 연결
- 배지·점심 여권·리더보드·월간 식당은 삭제가 아니라 2차 필수다.
## 15. 초기 데이터
- Kakao API로 기본 식당을 확보한다.
- 메뉴·가격·영업시간은 CSV로 일괄 등록하고 관리자 검증 후 반영한다.
- Kakao 장소 ID, 누락, 숫자 형식, 중복, 신규/수정 항목을 검사한다.
- 초기 목표는 자주 갈 식당 10~20곳이다.
- 반영 후 관리자 화면에서 수정할 수 있다.
## 16. 관리자 기능
- 사용자, PIN 초기화, 비활성화, 초대코드 변경
- 식당 활성/비활성, 추천 제외, 대표 사진
- CSV 업로드·검증, 메뉴·가격·영업시간 수정과 복구
- 리뷰·사진 삭제, 신고 처리, 공지, 회사 좌표·기본 반경, 관리자 로그
## 17. 1차 MVP 필수 범위
- 프로필/세션, 관리자 인증, Kakao 식당 수집
- 목록·지도·검색·필터·정렬, 식당 상세
- 메뉴·가격·영업시간 편집, 사진, 상태 제보
- 빠른/조건 추천, 당일 추천 제외
- 여기로 결정, 개인 방문, 직접/링크 초대, 수락·거절, 약속 변경
- 방문 확인, 먹은 메뉴·가격, 리뷰, 단순 도감
- 내부 알림, 신고, CSV, 관리자, 수정 이력
## 18. 2차 필수 범위
- 점심 투표방, 실시간 득표와 투표 변경, 참여자 후보 추가, 자동 마감, 동점 룰렛
- 공개 동행 모집, `함께 먹기` 하단 메뉴
- 배지, 점심 여권, 월간 리더보드, 이번 달의 식당, 개인 월간 결산, 시즌 배지
- 리뷰 도움돼요, 도감 수집 요소 확장, Supabase Realtime 투표 갱신
## 19. 명시적 제외
- 직원 이메일·휴대폰·소셜 로그인, 사용자 GPS, 여러 회사·지점
- 개인 기본 예산, 투표 예산 병합, 알레르기·채식·매운맛 프로필
- 주간 미션, 실시간 채팅, 리뷰 댓글, 사진 댓글·반응
- 외부인 참여, 지속 지도 크롤링, 이메일·문자 알림, 금전 보상
## 20. 구현 원칙
- 전체 기능을 한 번에 구현하지 말고 작동하는 수직 기능 단위로 진행한다.
- 각 단계는 UI, 서버 검증, DB, 테스트까지 완결한다.
- 기존 코드를 먼저 읽고 현재 구조를 따른다.
- 불필요한 추상화, 대규모 리팩터링, 요구되지 않은 기능 추가를 금지한다.
- 기획을 임의 변경하지 않는다.
- 진행 가능한 모호함은 합리적으로 가정하고 보고한다.
- 데이터 손실·보안·비용 관련 결정만 작업 전 질문한다.
- DB 제약조건과 서버 권한 검증을 함께 사용하고 모든 입력을 서버에서 검증한다.
- 날짜·시간은 Asia/Seoul 기준으로 처리한다.
- 실패·빈 데이터·중복 요청을 정상 흐름으로 설계한다.
- 비밀 값과 실제 초대코드를 커밋하지 않는다.
- 완료 전 lint, typecheck, test, build를 실행한다.
## 21. Claude 작업 절차
1. 저장소와 이 파일을 읽는다.
2. 현재 구현 상태와 관련 파일을 조사한다.
3. 수정 전에 작은 작업 계획을 제시한다.
4. 이번 범위와 제외 범위를 명시한다.
5. 승인받은 범위만 구현한다.
6. 테스트와 빌드를 실행한다.
7. 변경 파일, 결정, 검증 결과, 남은 작업을 보고한다.
- 새 프로젝트라도 바로 전체 코드를 생성하지 않는다.
- 먼저 프로젝트 구조, 환경변수, DB 스키마, 라우트, 구현 단계, 테스트 전략을 제안한다.
